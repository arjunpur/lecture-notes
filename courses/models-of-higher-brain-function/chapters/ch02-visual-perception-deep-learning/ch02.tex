\section{Lecture 1: Visual Perception and Deep Learning}

These notes cover the opening perception lecture of \emph{Models of Higher Brain Function}. The lecture uses vision as the first concrete example of a higher-brain computation: the sensory input is high-dimensional, noisy, and ambiguous, but behavior requires a stable internal description of objects and scenes. Deep neural networks enter the lecture not as a fashionable add-on, but as a way to make the representation problem explicit. They give us a concrete family of models that can solve large-scale object recognition tasks and can then be compared with the ventral visual stream.

\subsection{Why vision belongs in a course on higher brain function}

The lecture opens with a basic question: what does a visual system need to know about the world? At the level of behavior, the answers are simple enough:
\begin{itemize}
    \item what is out there,
    \item where it is,
    \item and what it is likely to do next.
\end{itemize}

Those questions already show why vision is not just a matter of registering light intensity. A retinal image is only a pattern of local measurements. Perception requires the system to organize that pattern into useful latent variables such as object identity, position, motion, and behavioral relevance.

\subsubsection{Why this problem is harder than it feels}

One theme from the lecture is Moravec's paradox: tasks that feel effortless to humans often turn out to be extremely demanding computationally. Recognizing a cup after it has moved, rotated, changed scale, or appeared under different illumination feels trivial in everyday life, but it is difficult because the raw sensory input changes enormously while the object identity stays the same.

This is why visual perception is treated here as a problem of inference rather than mere detection. The brain must move from a changing image to a useful internal representation of the world.

\subsection{Invariant object recognition}

The core challenge identified in the lecture is \emph{invariant object recognition}. A useful visual system should keep track of what an object is even when many superficial aspects of the image change.

\subsubsection{Why pixels are a bad format for recognition}

The lecture follows the argument from DiCarlo and Cox that the raw image is a poor coordinate system for object recognition. Pixel values are highly sensitive to:
\begin{itemize}
    \item translation across the retina,
    \item viewing angle,
    \item distance and scale,
    \item illumination,
    \item occlusion,
    \item and background clutter.
\end{itemize}

So two pictures of the same object can be very far apart in pixel space, while pictures of different objects may overlap heavily in their low-level image statistics. The slides describe this by saying that object manifolds are ``hopelessly tangled up'' in the original sensory coordinates.

\subsubsection{What makes a representation useful}

The lecture's answer is that a ``good'' neural representation should vary gradually with nuisance variables such as position, illumination, or angle, while making object identity easier to read out. In other words, recognition becomes easier when the representation performs a kind of \emph{disentangling}: factors that should be treated as irrelevant for identification have a smaller effect on the code than they do in the original image.

This idea is important because it shifts the focus away from raw accuracy and toward representational geometry. The real question is not only whether the system labels an image correctly, but whether it transforms sensory input into coordinates that make the task easy for later stages.

\subsection{The ImageNet turning point}

The lecture then turns from the abstract recognition problem to the modern deep-learning solution. The key historical moment is the ImageNet era. ImageNet supplied a large-scale benchmark with roughly fifteen million labeled images spanning one thousand categories. This mattered because it forced models to solve recognition at a realistic visual scale rather than on small toy datasets.

\subsubsection{Why AlexNet changed the conversation}

AlexNet's 2012 ImageNet result is presented as the turning point. Its top-5 error was \(16.4\%\), while the next-best system remained at \(26.2\%\). The significance of that result was not just the leaderboard gap. It showed that a deep, trainable convolutional network could learn useful visual features directly from data instead of depending mainly on hand-engineered descriptors.

The lecture also describes the ``magic sauce'' behind this success. Several ingredients had to come together at once:
\begin{itemize}
    \item large labeled datasets,
    \item data augmentation,
    \item substantial computing power,
    \item convolution and weight sharing,
    \item rectified linear units,
    \item pooling and downsampling,
    \item and architectures deep enough to build hierarchical features.
\end{itemize}

So deep learning did not succeed because of one isolated trick. It succeeded because data, optimization, hardware, and architecture started to reinforce one another.

\subsubsection{From AlexNet to ResNet}

The lecture places AlexNet in a broader architectural sequence. VGG-style networks increased depth and regularized the use of small convolutions. ResNets then made very deep models trainable by adding skip connections. Instead of asking each block to learn an entirely new transformation, a residual block can learn a correction to the identity map. This helps with vanishing gradients and with the degradation in performance that appears when very deep plain networks become hard to optimize.

The slides make this concrete with the ImageNet benchmark again: ResNets reached depths above one hundred layers and pushed error rates below typical human performance on that dataset. The lecture's point is not that ImageNet is a perfect model of human vision, but that these models had become competent enough that their internal representations became scientifically interesting.

\subsection{What deep networks compute}

At the mathematical level, the lecture describes a deep network as a concatenation of linear-nonlinear transformations. A single layer takes an input vector \(\vec{x}\), applies an affine transformation, and then a nonlinearity:
\[
\vec{y} = \phi\!\left(W\vec{x} + \vec{b}\right).
\]
Here \(W\) is the weight matrix, \(\vec{b}\) is a bias term, and \(\phi\) is a nonlinear activation function.

\subsubsection{Why nonlinearity matters}

If every layer were purely linear, then stacking many layers would still amount to one overall linear transformation. Depth would not buy us much representational power. Nonlinearity is what lets the network bend and reshape the input space so that later layers can express more abstract and more separable features.

The lecture's qualitative story is straightforward:
\begin{itemize}
    \item early layers detect simple local patterns such as oriented edges or contrast changes,
    \item intermediate layers combine those into textures or parts,
    \item later layers become more useful for category-level distinctions.
\end{itemize}

This staged abstraction is exactly why deep networks became plausible models of a hierarchical sensory pathway.

\subsubsection{Why convolution is useful for images}

Convolutional layers add two inductive biases that fit images well.

\paragraph{Local receptive fields.}
Each filter only sees a small image patch at a time. This reflects the fact that nearby pixels are strongly related and that many informative visual structures are local.

\paragraph{Weight sharing.}
The same filter is applied at many spatial positions. This makes it possible to detect the same local pattern wherever it appears and reduces the total number of parameters that must be learned.

Pooling and striding then expand the effective receptive field and make the representation less sensitive to small translations. This does not create perfect invariance, but it pushes the representation in the right direction for object recognition.

\subsubsection{Backpropagation and the credit-assignment problem}

Training is described in the lecture through backpropagation and stochastic gradient descent. A loss function measures the discrepancy between the network output and the target label, and gradients are propagated backward through the chain of nonlinear functions. A typical parameter update has the form
\[
w \leftarrow w - \eta \frac{\partial \mathcal{L}}{\partial w},
\]
where \(\mathcal{L}\) is the loss and \(\eta\) is the learning rate.

This solves the engineering problem of \emph{credit assignment}: it tells the system how each parameter contributed to the final error. The lecture is careful, however, not to overinterpret this as a biological mechanism. Exact backpropagation, large labeled datasets, and perfectly shared convolutional filters are all biologically questionable. So deep networks are useful models of visual computation without being literal copies of cortical learning.

\subsection{Deep networks as models of ventral visual cortex}

Once deep networks became good at object recognition, the next question was whether their internal representations looked at all like those found in ventral visual cortex. The lecture emphasizes that the comparison should not be made unit by unit. A hidden unit in a trained network is not expected to correspond neatly to one biological neuron, and different random initializations of the same architecture need not preserve any unique one-to-one matching.

\subsubsection{Why compare representational geometry}

The more useful comparison is at the level of \emph{representational similarity}. For a collection of stimuli, one can compute a response pattern for each stimulus and then ask how dissimilar those patterns are from one another. This yields a representational dissimilarity matrix:
\[
\mathrm{RDM}_{ij} = 1 - \mathrm{corr}(r_i, r_j).
\]
Here \(r_i\) and \(r_j\) denote the response vectors evoked by stimuli \(i\) and \(j\).

The important point is that two systems can have similar representational geometry even if their individual units are not aligned. This is why representational similarity analysis became so useful in comparing human IT, monkey IT, and deep-network hidden layers.

\subsubsection{What the comparisons show}

The lecture highlights a consistent empirical pattern: supervised deep models tend to explain the geometry of IT responses better than unsupervised alternatives, and models that are better optimized for object recognition often provide better predictions of neural data. This does not mean that cortex literally implements AlexNet or ResNet. It means that solving the right task appears to push artificial systems toward internal representations that partially resemble those found in the brain.

That is a strong scientific claim. It suggests that task demands and representational constraints are deeply connected: if a system must solve invariant object recognition well, some aspects of its internal geometry may be forced into a similar shape, whether the system is biological or artificial.

\subsection{Deep networks as experimental tools}

The lecture goes further than model comparison and treats deep networks as practical tools for neuroscience experiments. The Bashivan-style examples make this especially clear. The logic is:
\begin{enumerate}
    \item train a convolutional network on a large visual task,
    \item learn a mapping from model features to recorded neural responses,
    \item optimize the input image to maximize the predicted response of a chosen neuron or population,
    \item and then test the synthesized image experimentally.
\end{enumerate}

This matters because a differentiable model can be used not only to \emph{explain} old data but also to \emph{design} new stimuli. If the model captures something important about a neuron's preferred feature geometry, then the optimized image should strongly drive that neuron. The lecture discusses V4 experiments in which CNN-optimized images indeed strongly activated recorded neural sites.

So the deep model becomes more than a classifier. It becomes a hypothesis generator for what features a neural population is actually sensitive to.

\subsection{What deep networks still miss}

The lecture closes with several cautions. Modern deep networks are highly effective at image classification, but that does not make them complete theories of biological vision.

\subsubsection{Adversarial fragility}

Deep networks can be fooled by adversarial perturbations: small image changes that are nearly imperceptible to humans but cause a dramatic change in the predicted label. This shows that strong benchmark performance does not automatically imply human-like robustness or understanding.

\subsubsection{Biological caveats}

Several ingredients of standard deep learning remain biologically questionable:
\begin{itemize}
    \item exact backpropagation,
    \item extensive supervised labels,
    \item strict convolutional weight sharing,
    \item and mostly feedforward inference in systems whose biological counterparts are strongly recurrent.
\end{itemize}

These caveats do not make deep networks unhelpful. They simply define the proper level of interpretation. Deep networks are useful computational models of visual recognition, and they may capture important aspects of cortical representation, but they are still abstractions.

\subsection{Conceptual takeaways from Lecture 1}

The opening vision lecture establishes the framework used later in the course.

\begin{itemize}
    \item The main difficulty of vision is not detecting light, but constructing a representation that supports stable object recognition.
    \item Raw images are a bad format for recognition because nuisance variables strongly distort pixel values.
    \item Deep networks matter because they provide a concrete way to learn hierarchical feature spaces that partially solve this representation problem.
    \item The success of deep learning depends on a combination of architecture, data, optimization, and compute rather than on one isolated trick.
    \item Model evaluation can proceed on several levels at once: recognition performance, similarity to cortical representations, and usefulness for generating new experiments.
    \item High benchmark accuracy is not the same as biological plausibility or human-like robustness.
\end{itemize}

These points motivate the broader theme of the course: higher brain function can often be studied by asking what representation a system needs in order to solve a demanding cognitive task.
